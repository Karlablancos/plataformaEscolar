# Changelog

Carpeta reservada para registrar cambios futuros del proyecto.

## Convencion sugerida

Crear un archivo Markdown por cambio relevante o release.

Formato de nombre sugerido:

```text
YYYY-MM-DD-descripcion-corta.md
```

Ejemplo:

```text
2026-06-09-documentacion-inicial.md
```

## Plantilla

```md
# Titulo del cambio

## Fecha

YYYY-MM-DD

## Componentes afectados

- `frontend`
- `bff-gateway`
- `usuario`
- `establecimiento`
- `asistencia`
- `database`

## Resumen

Descripcion breve del cambio.

## Cambios tecnicos

- Cambio 1.
- Cambio 2.

## Migraciones o configuracion

- Indicar cambios de base de datos, variables de entorno o configuracion.

## Pruebas

- Indicar pruebas ejecutadas.
```
